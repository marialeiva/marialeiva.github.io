OBJETIVE

The purpose of this document is to provide guidance, information, and advice of good programming practices to maintain a high quality of development in order to increase the rate of successful RDA macros development.

SCOPE

Create standards for developing VBA Macros and set up policies and rules that improve the quality and performance of the process. This will provide a way to develop and maintain a macro effortlessly.

GOOD PRACTICES

1. Modular Structure

The structure must be easy to understand for other developers. Use a "black box" approach where each function performs a specific task. Related procedures should be grouped in modules.

' Example of Modular Main Structure Sub Main() Call InitialSettings() ' Prep environment Call OpenSourceFiles() ' Load data Call BusinessLogic() ' Core processing Call FinalizeAndSave() ' Output End Sub

2. English Documentation

For global compatibility, all internal documentation (variable names, comments, procedure titles) must be written in English.

3. Performance Features

Mandatory optimization to increase speed and efficiency by disabling UI overhead during macro execution.

' High-Performance Block With Application .ScreenUpdating = False .Calculation = xlCalculationManual .EnableEvents = False .DisplayAlerts = False End With

4. Recording Macros

Recorded macros are for reference only. Production code must be cleaned. Remove redundant commands like .Select or .Activate.

6. Working with SharePoint

Avoid hardcoded paths. Use dynamic environment variables to ensure the macro works for any user on any synchronized station.

' Dynamic Pathing strPath = Environ("USERPROFILE") & "\SharePoint\Automation\Data\"

7. Working with SAP & Outlook

Always validate SAP Scripting status and Outlook account visibility. Use .Display during testing and .Send only for production.

' Outlook Account Handling For Each oAcc In OutApp.Session.Accounts If UCase(oAcc) = UCase(strSender) Then With objMail Set .SendUsingAccount = oAcc .HTMLBody = strBody & .HTMLBody .Display End With End If Next
